# BP-006 — `RouteGroupMiddleware` reads `req.url` (mount-relative) instead of `req.originalUrl`

**Type:** bug
**Severity:** high — `skipAuth: true` public route groups never detected
**Affects:** `@rhino-dev/rhino-nestjs@0.1.0`

## Symptom

Config:

```ts
routeGroups: {
  auth: {
    prefix: 'auth',
    models: [],
    skipAuth: true,
  },
}
```

Intent: any request whose URL contains `/auth/` should set `req.__skipAuth = true` so `JwtAuthGuard` short-circuits.

Actual: every request to `/api/auth/login` still gets `{ code: 'UNAUTHORIZED', message: 'Missing bearer token' }`. The middleware is running, but `req.url` inside the middleware is `/` (root — the path after Express has stripped the mount prefix where the middleware is attached with `forRoutes('*')`).

## Root cause

`src/middleware/route-group.middleware.ts` (library):

```ts
use(req, _res, next) {
  const url = String(req.url ?? '').split('?')[0];
  const groups = this.config.routeGroups();
  for (const [name, group] of Object.entries(groups)) {
    const prefix = group.prefix ?? '';
    if (!prefix) continue;
    if (prefix.startsWith(':')) continue;
    if (url.includes(`/${prefix}/`) || url.endsWith(`/${prefix}`)) {
      match = { name, prefix };
      if (group.skipAuth) req.__skipAuth = true;
      break;
    }
  }
  ...
}
```

In Express, `req.url` is path MINUS the current mount point. When `consumer.apply(RouteGroupMiddleware).forRoutes('*')` is used, each middleware handler is mounted at the route it's being called for. Express strips the mount path — so for a request to `/api/auth/login`, inside the middleware `req.url` looks like `/` (or the remaining path after the middleware's implicit mount).

`req.originalUrl` always contains the full request URL.

## Proof

I installed a debug middleware:

```ts
@Injectable()
export class DebugMiddleware implements NestMiddleware {
  use(req, _res, next) {
    console.log(`${req.method} ${req.url} origUrl=${req.originalUrl}`);
    next();
  }
}
```

Output for `curl http://localhost:8004/api/auth/login`:
```
POST / origUrl=/api/auth/login
```

`req.url === '/'` → `'/'.includes('/auth/')` → false → `__skipAuth` never set.

## Fix

One-line change in the library:

```ts
- const url = String(req.url ?? '').split('?')[0];
+ const url = String(req.originalUrl ?? req.url ?? '').split('?')[0];
```

Same fix needed in `ResolveOrganizationMiddleware` — it reads `req.params.organization` which DOES work because Nest wires params separately, so that one is OK. But any middleware that needs to see the full path must use `originalUrl`.

## Workaround

`src/public-paths.middleware.ts` in the template reads `req.originalUrl` directly and hard-codes the `/api/auth/` and `/api/invitations/accept` prefixes. Template sets `autoRouteGroupMiddleware: false` and replaces with its own middleware.

## Acceptance criteria for fix

- [ ] `req.originalUrl` used in `route-group.middleware.ts` url resolution.
- [ ] Unit test: mock request with `originalUrl: '/api/auth/login'` + route group with `prefix: 'auth', skipAuth: true` → `req.__skipAuth === true`.
- [ ] E2E test in the library: public route group works with the built-in `JwtAuthGuard`.
- [ ] Template can drop `PublicPathsMiddleware` and re-enable `autoRouteGroupMiddleware: true`.
